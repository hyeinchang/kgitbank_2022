
1. [ git clone ] 

1. clone : github에 연결되지 않은 상태에서 내려받음
------------------------------------------------------------------------------------------
* pull : github에 연결된 상태에서 내려받음
* 새로운 공간에서 사용
------------------------------------------------------------------------------------------


2. git clone <원격지 주소> : 해당 원격지의 repository 데이터를 내려받음
------------------------------------------------------------------------------------------
$ git clone https://github.com/hyeinchang/test4.git

2. [ githun branch ]

1. git branch -r : 원격지의 branch를 보여줌
------------------------------------------------------------------------------------------
$ git branch -r
  origin/master
------------------------------------------------------------------------------------------


2. git branch -a : repository와 원격지의 branch를 보여줌
------------------------------------------------------------------------------------------
$ git branch -a
  a-branch
* master
  remotes/origin/master
------------------------------------------------------------------------------------------


3. git push <원격지 이름> <브런치 이름> : 해당 원격지에 해당 브런치를 추가
------------------------------------------------------------------------------------------
$ git push origin a-branch
Total 0 (delta 0), reused 0 (delta 0), pack-reused 0
remote:
remote: Create a pull request for 'a-branch' on GitHub by visiting:
remote:      https://github.com/hyeinchang/test07/pull/new/a-branch
remote:
To https://github.com/hyeinchang/test07.git
 * [new branch]      a-branch -> a-branch
------------------------------------------------------------------------------------------


4. git push <원격지 이름> --delete <브런치 이름> : 해당 원격지의 해당 브런치 삭제
------------------------------------------------------------------------------------------
$ git push origin --delete a-branch
To https://github.com/hyeinchang/test07.git
 - [deleted]         a-branch
------------------------------------------------------------------------------------------


5. git push <원격지 이름> <원격지에 추가한 브런치 이름> : 원격지에 추가한 해당 브런치로 commit한 내용을 올림
------------------------------------------------------------------------------------------
$ git push -u origin master

------------------------------------------------------------------------------------------


6. git pull 할 경우 해당 원격지의 checkout 되어 있는 해당 브런치의 내용을 가져옴


7. git clone을 사용하여 원격지의 repository를 복사했을 경우 원격지에 있는 브런치를 가져옴으로 원격지의 브런치들로 chekout 가능 
------------------------------------------------------------------------------------------
$ git branch -a
* master
  remotes/origin/HEAD -> origin/master
  remotes/origin/a-branch
  remotes/origin/master

$ git checkout a-branch
Switched to a new branch 'a-branch'
branch 'a-branch' set up to track 'origin/a-branch'.
------------------------------------------------------------------------------------------
