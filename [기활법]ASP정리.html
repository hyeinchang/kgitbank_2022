1. ASP

1) ASP란 무엇인가?

  • Microsoft에서 개발한 웹페이지 개발 환경
  • IIS 3.0부터 도입됨(Window NT SP3)
  • Jscript 또는 VBScript가 주요 구성요소임
  • ADO 컴포넌트를 사용하여 DB연동이 쉬움

2) ASP의 특징 및 장점

  • DB나 외부 컴포넌트를 불러들여 웹서버에 기능을 추가할 수 있음
  • HTML코드에 비해 안점함
  • 동적인 페이지를 구성할 수 있음
  • Windows 플랫폼에 종속적임
  • 스크리팅 언어(VBScript/JavaScript)로 프로그래밍 됨
  • 웹브라우저와 관계없이 사용할 수 있음

3) ASP 처리방식

○ 웹브라우저/웹서버/ASP/DBMS의 연동 처리과정
웹브라우저(IE) ←→ 웹서버(IIS) ←→ ASP ←→ DBMS(MS-SQL)

4) ASP의 구성

○ HTML
- 클라이언트로부터 다양한 정보를 받아들이기 위한 입력 폼을 만들기 위해 사용
- 클라이언트가 요청한 결과의 출력 양식을 지정하기 위해 사용
○ VBScript/Jscript
- 웹서버에서 클라이언트의 요청을 처리하기 위해서 사용하는데, ASP에서 사용가능한 스크립트는 VBScript, JScript등이 있으며, 이중에서 VBScript가 기본으로 사용
○ ASP 내장객체
- ASP는 COM 컴포넌트 중에서 웹 기반의 애플리케이션을 개발하기 위한 필수 구성요소로서 ASP 내장객체(Built-In Object)를 제공
- ASP 내장객체는 다른 COM 컴포넌트와 달리 인스턴스의 생성없이 사용가능하다는 장점이 있음
- ASP 내장객체로 구현할 수 없거나 어려운 기능에 대해서는 ADO 컴포넌트 및 외부 컴포넌트를 이용하여 구현
○ 객체들의 역할
- Request : 클라이언트 요청(HTTP Request)에 관련된 정보를 저장하는 객체
- Response : 서버 응답에 관련된 정보를 저장하는 객체
- Session : 클라이언트 세션에 관련된 정보를 저장하는 객체
- AspError : 오류상태에 관련된 정보를 저장하는 객체
- Server : 서버에서 사용 가능한 메소드와 프로퍼티의 접근 지원하는 객체
- Application : 웹 애플리케이션에 관련된 정보를 저장하는 객체
- ObjectContext : 트랜잭션 처리를 위해서만 사용하는 객체
○ ADO 객체
- 데이터베이스에 접근하기 위한 인터페이스를 제공하는 객체
○ 외부 COM 컴포넌트
- IIS COM 컴포넌트를 의미하며 스크립트 구문으로 처리할 수 없는 데이터베이스 접근, 트랜잭션 처리, 클라이언트 요청에 대한 차별화된 결과 전송 등의 작업을 처리

4. ASP 내장 객체 - Request 객체

Request 객체는 서버로 전송된 클라이언트 정보를 저장하는 역할을 함

1) 데이터 전송방식

  • 클라이언트가 웹서버로 데이터를 전송하는 방식은 GET 방식과 POST 방식이 있음
  • <Form method="GET | POST"> 태그를 사용하여 지정할 수 있음
○ GET 방식
- 클라이언트가 작성한 내용을 환경 변수에 저장하여 웹서버로 전송
- 웹서버는 Request 객체의 QueryString 컬렉션에 저장하여 값을 사용가능
- 웹브라우저의 주소창에 정보가 노출되기 때문에 보안상 위험이 있고, 웹서버로 전송 가능한 데이터 길이에 한계가 있다는 단점이 있음
○ POST 방식
- 클라이언트가 작성한 내용을 표준입력을 통해서 웹서버로 전송
- 웹서버는 Request객체의 Form컬렉션에 저장하여 값을 사용가능
- POST 방식을 이용하는 경우에 웹 브라우저의 주소창에는 URL만 표시되고, 클라이언트 정보는 HTTP요청 패킷에 포함되어 웹서버로 전송되기 때문에 보안성에 좋음
- 표준입력 방식을 사용하므로 데이터 길이에 제한이 없음
구분 GET 방식 POST 방식
전송 환경변수 표준입력
저장 Request.QueryString 컬렉션 Request.Form 컬렉션
데이터길이 제한함 제한없음

2) Request 객체의 Collection

  • Request 객체는 5개의 컬렉션을 사용하여 클라이언트로부터 전송된 다양한 정보를 분류하여 저장
  • 컬렉션을 사용할 때는 Request, 컬렉션이름(변수)와 같이 사용

3) Form 컬렉션

  • POST 방식으로 전송한 클라이언트 정보 저장

○ Property

프로퍼티 역할
ClientCertificate 컬렉션 클라이언트에서 웹서버로 전송한 인증서 필드 값 저장
Cookie 컬렉션 클라이언트의 HTTP요청 헤더에 포함된 쿠키값 저장
Form 컬렉션 POST방식으로 전송한 Form 요소 값 저장

PATH_TRANSLATED

QUERY_STRING
GET방식으로 데이터를 웹서버로 넘길때 URL뒤에 따랄붙는 전송내용을 나타냄

REMOTE_ADDR
클라이언트의 IP주소를 나타냄
REMOTE_USER
사용자 이름을 나타냄
REQUEST_METHOD

SCRIPT_NAME
현재 실행이 요청된 스크립트명을 나타냄
SERVER_SOFTWARE
웹서버의 이름과 버전을 나타냄
SERVER_NAME

URL
서버의 URL을 나타냄
SERVER_PROTOCOL
클라이언트 요청시 사용되는 프로토콜을 나타냄
SERVER_PORT
사용할 TCP/IP 포트를 나타냄
HTTP_REFERER
이전 URL을 나타냄
REQUEST_URI
URI를 나타냄
APPL_PHYSICAL_PATH
프로젝트의 물리적위치를 나타냄
PATH_TRANSLATED

7. Server 객체

Server 객체는 서버에서 접근할 수 있는 메소드와 프로퍼티를 사용할 수 있도로 지원하는 객체

메소드 역할
CreateObject 서버 컴포넌트의 인스턴스를 생성
MapPath 웹서버의 가상 또는 상대 경로를 절대 경로로 변환
Exceute *.asp 파일을 실행하고 결과반환
Transfer *.asp 파일 실행

프로퍼티 역할
ScriptTimeout 스크립트를 처리하기 위한 최대 시간

1) CreateObject 메소드와 인스턴스

  • CreateObject 메소드는 서버 컴포넌트의 인스턴스를 생성
  • ASP 객체는 사용할 때 별도의 인스턴스 생성과정없이 바로 사용할 수 있지만, 서버내의 컴포넌트(*.dll 파일 등)는 인스턴스를 생성해야 사용할 수 있음 (서버 컴포넌트를 ASP 스크립트 해석기가 직접 해석할 수 없기 때문)
  • 인스턴스란 클래스 형태로 정의해 놓은 객체를 실제로 사용할 수 있는 형태 즉, 객체의 메소드와 프로퍼티를 활용할 수 있는 상태로 만들어 놓은 것을 의미

2) 인스턴스 생성

  • Set 인스턴스명=Server.CreateObject("클래스ID | 프로그램ID")
  • 사용하고자 하는 컴포넌트의 클래스ID 또는 프로그램ID로 인스턴스를 생성

3) 인스턴스 해제

  • 생성된 인스턴스에는 서버 리소스가 할당되기 때문에 인스턴스를 사용한 후에는 반드시 서버 리소스를 반환해줘야 하는데, 그 방법은 'Set 인스턴스명 = nothing'이라는 문장을 스크립트의 마지막 부분에 추가

ex) CreateObject 메소드 사용

<%

set DBConn = Server.CreateObject("ADODB.Connenction")

DBConn.open "shop", "sa", "1111"

sql = "SELECT * FROM member"

set rs = DBConn.Execute(sql)

set rs = Nothing

DBConn.Close

set DBConn = nothing

%>

    ...
    </Script>

12. ASP 내장 객체 - Session 객체

  • 서버에 접속하는 각 클라이언트 마다 개별적인"세션 연결 정보"를 저장

○ Method / Property / Event

메소드 역할
Abandon Session 객체에서 사용하고 있는 모든 자원을 해제

프로퍼티 역할
Contents 컬렉션 세션 범위의 변수를 저장

이벤트 역할
Session_OnStart 세션이 연결될 때 실행
Session_OnEnd 세션이 끊어지거나, 연결 시간이 초과되면 실행

1) Session 변수

  • 클라이언트의 연결 정보를 저장하는 것으로, 클라이언트가 페이지를 이동하더라도 삭제되지 않음
○ Session 변수 생성
- Session("변수") = 값

2) Contents 컬렉션

  • Session 객체의 Contents 컬렉션은 Session 객체에 저장된 모든 항목을 의미

<%

Session("GlobalDept") = "연습"

Session("GlobalEmail") = "user1@asptest.com"

Session("GlobalCount") = 12345

Response.Write i & "=" & Session.Contents(i) & "<br>"

NEXT

%>

○ Session 객체의 Contents 컬렉션 항목 삭제
Session("ACount") = 10
Session("BCount") = 20
Session.Contents.Remove("ACount")
Session.Contents.RemoveAll() 'Session 객체의 Contents 컬렉션의 모든 항목 삭제

3) SessionID 프로퍼티

  • 웹서버에 접속하는 클라이언트에 부여자되는 숫자로서, 각 클라이언트를 구별하기 위하여 사용
  • 클라이언트마다 다른 값을 가지며 쿠키로 저장되었다가 브라우저가 종료되거나 연결 제한 시간이 만료되면 자동 삭제
  • 웹 브라우저에서 쿠키 사용을 금지하면 SessionID를 저장할 수 없기 때문에 회원인증처리에서 문제 발생 가능

4) Timeout 프로퍼티

  • Session객체의 만료시간을 설정하기 위한 것으로, 스크립트에서 그 값을 설정가능
  • Timeout값은 너무 크면 서버의 성능 악화를 초래할 수 있고, 너무 작으면 잦은 연결설정으로 사용자의 불편을 초래할 수 있으므로 적절히 설정
  • Timeout프로퍼티의 기본값은 20(분)

5) Abandon 메소드

  • 서버와의 논리적 연결 개념인 세션을 강제적으로 종료하는 것으로, 회원제 사이트에서의 로그아웃 등에 사용가능
  • Abandon 메소드가 실행되면 Session객체의 모든 정보가 삭제되고 서버 리소스가 해제

ex) SessionID, Timeout, Abandon

  • Session 객체의 생성과 삭제는 Session 객체 이벤트와 함께 이루어지는데, Session객체 이벤트는 Session_OnStart 이벤트와 Session_OnEnd 이벤트로 구성

6) 배열 함수

함수 내용 예제
LBound (배열명, 차원) 배열의 첨자 하한값 반환
1차원은 LBound(배열명), 2차원은 LBound(배열명, 차원) 		Dim arr(5)
a = LBound(arr) → 0
Dim arr(5, 7)
a = LBound(arr, 2) → 0
UBound (배열명, 차원) 배열의 첨자 상한값 반환
1차원은 LBound(배열명), 2차원은 LBound(배열명, 차원) 		Dim arr(5)
a = UBound(arr) → 5
Dim arr(5, 7)
a = UBound(arr, 1) → 5
a = UBound(arr, 2) → 7

ex)

<%

Dim a(5)

a(1) = 1 : a(2) = 2 : a(3) = 3 : a(4) = 4 : a(5) = 5

For i=1 To UBound(a)

Response.Write a(i)

Next

%>
